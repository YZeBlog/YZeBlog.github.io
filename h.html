Hexo自动化服务架构设计

架构设计模型

graph TD
A[用户终端] --> B{Nginx代理层}
B --> C[静态文件服务]
B --> D[管理后台]
B --> E[API服务]
D --> F[权限验证]
D --> G[内容管理]
E --> H[文件操作]
E --> I[部署引擎]
I --> J[Git集成]
I --> K[生成监控]

架构拓扑图

            graph LR
            U[用户] --> N{Nginx 80/443}
            N --> S[静态站点 /var/www/hexo]
            N --> A[管理后台 :3000]
            N --> API[API服务 :8000]
            API --> H[Hexo Core]
            H --> G[Git仓库]
            H --> FS[文件系统]
            A --> DB[(SQLite)]
            API --> MQ[消息队列]
            MQ --> W[Worker进程]

架构服务流程

内容创作

管理员通过Web编辑器创建Markdown文档

版本控制

API服务触发Git提交操作，保存变更历史

静态生成

Hexo生成引擎执行generate命令构建站点

部署发布

Nginx实时加载新生成的静态文件

架构深度解析

安全架构设计
  • 三层权限隔离：系统用户隔离、容器隔离、应用级RBAC
  • 操作沙箱机制：危险命令在独立线程执行
  • HTTPS全链路加密
高可用设计
  • Nginx负载均衡：双活节点部署
  • 状态分离设计：/source目录独立存储
  • 异常恢复机制：自动回滚最后可用版本
